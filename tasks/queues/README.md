# Очереди статусов API задач

Единый пул статусов: `queued`, `in_progress`, `ready_for_spec`, `review`, `completed`, `archived`, `cancelled`.

- Каждая очередь ведётся в отдельном файле `*.md` с лимитом 500 строк.  
- При переполнении создавайте продолжения (`<status>_0001.md`, `<status>_0002.md`) и добавляйте ссылку на следующую часть в начале файла.  
- При смене статуса переносите запись между файлами, предотвращая дублирование.  
- Используйте единый формат записи (см. примеры внутри каждого файла).
